Pragmatic Hybrid Entity Component System

理論よりも実用を ― Component指向とECSの融合設計

Introduction

Pragmatic Hybrid Entity Component System（PH-ECS） は、 ゲーム開発において「純粋なECSでは得られない柔軟性」と 「ECSの分割する考えによるコードの明白さ」を両立させるために設計された 実用的ハイブリッドアーキテクチャ です。

このエンジンは、コンポーネントを単なるデータ構造ではなく、 データ構造＋補助メソッドとして扱います。

Concept – Pragmatic & Hybrid

1. Pragmatic（実用主義）

純粋ECSの理論美よりも「現場で使える設計」を優先。 コンポーネントはデータのみでなく、Getter / Setter / RenderImGui / Serialize / Deserializeのような 補助的な動作関数を持つことを容認します。
類似の設計として tiny-ecsがあります。

2. Hybrid（複合型）

内部はECS構造を採用しながら、エディタではコンポーネント指向的UIを提供。 開発者は「ECSとして実行し、コンポーネントとして編集する」という 双方向のワークフローを得られます。
同義の設計として Unity ECSがあります。

3. Editor Integration

すべてのECS要素はエディタと強く統合され、 コンポーネント追加・削除・DLLホットリロード・Pythonプラグイン実行などを 一元管理可能です。
